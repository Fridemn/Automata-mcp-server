## 服务概览

Automata MCP Server 是一个基于 FastAPI 的 MCP (Model Context Protocol) 服务器，提供两种接入方式：

| 接入方式 | 端点 | 适用场景 |
|----------|------|----------|
| MCP 协议 | `POST /mcp` | AI 客户端（Claude Desktop、n8n MCP 节点等） |
| REST API | `POST /tools/{name}` | 自定义应用、脚本、前端集成 |
| OpenAPI | `GET /docs` / `GET /openapi.json` | 浏览、调试、生成客户端 SDK |

两种方式调用的是同一套工具，MCP 协议本质上是对 REST API 的封装——`fastapi-mcp` 库将所有 FastAPI 路由自动转换为 MCP 工具定义，调用时内部转发到对应的 REST 端点。

## 认证方式

所有工具端点均需要 Bearer Token 认证。

### 配置 Token

通过环境变量 `AUTOMATA_ACCESS_TOKEN` 设置：

```bash
export AUTOMATA_ACCESS_TOKEN="your-secure-token"
```

或在 `.env` 文件中：

```env
AUTOMATA_ACCESS_TOKEN=your-secure-token
```

### 传递 Token

**REST API** 请求通过 `Authorization` 头传递：

```
Authorization: Bearer your-secure-token
```

**MCP 协议** 客户端在连接配置中填写 Access Token，客户端会自动将其作为 Bearer Token 发送。

### 注意事项

- Token 长度至少 8 个字符
- 如果未配置 `AUTOMATA_ACCESS_TOKEN`，服务端会打印警告并**允许所有请求通过**（仅限开发环境）
- 使用 `hmac.compare_digest()` 进行恒定时间比较，防止计时攻击

### 免认证端点

以下路径无需认证：

| 路径 | 说明 |
|------|------|
| `GET /` | 服务信息 |
| `GET /health` | 健康检查 |
| `GET /docs` | Swagger UI |
| `GET /redoc` | ReDoc 文档 |
| `GET /openapi.json` | OpenAPI Schema |
| `GET /static/*` | 静态文件 |
| `GET /api/logs` | 日志查看页面 |

## 方式一：MCP 协议接入

适用于支持 MCP 协议的 AI 客户端，如 Claude Desktop、n8n MCP 节点等。

### Streamable HTTP 传输

服务端使用 `fastapi-mcp` 库的 **Streamable HTTP** 传输模式，端点为：

```
http://<host>:<port>/mcp
```

该端点支持 `GET`、`POST`、`DELETE` 方法，由 `StreamableHTTPSessionManager` 管理 MCP 会话。

### 接入 Claude Desktop

在 Claude Desktop 的配置文件中添加：

**macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "automata": {
      "url": "http://localhost:8000/mcp",
      "headers": {
        "Authorization": "Bearer your-secure-token"
      }
    }
  }
}
```

> 如果 Claude Desktop 尚不支持 Streamable HTTP 传输，可联系服务端管理员启用 SSE 传输（端点为 `/sse`）。

### 接入 n8n

在 n8n 中添加 MCP Server 节点时：

1. **Transport Type**: 选择 Streamable HTTP
2. **Server URL**: `http://<host>:<port>/mcp`
3. **Access Token**: 填写 `AUTOMATA_ACCESS_TOKEN` 的值

> **注意**：旧版文档中提及的 `/sse` 端点当前未启用。如需 SSE 传输，需在服务端额外调用 `self.mcp.mount_sse()`。

## 方式二：REST API 接入

适用于自定义应用、脚本或前端集成。

### 调用工具

所有工具通过 `POST /tools/{tool_name}` 调用。`tool_name` 对应工具目录名（snake_case）。

#### JSON 请求

大多数工具使用 JSON 请求体：

```bash
curl -X POST http://localhost:8000/tools/fetch \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-secure-token" \
  -d '{
    "url": "https://example.com",
    "max_length": 5000
  }'
```

#### 表单请求（文件上传）

支持文件上传的工具使用 `multipart/form-data`：

```bash
curl -X POST http://localhost:8000/tools/image_upload \
  -H "Authorization: Bearer your-secure-token" \
  -F "file=@/path/to/image.png"
```

> 具体某个工具使用 JSON 还是表单，取决于工具的 `use_form` 配置。可查阅 `/docs` 页面或 `GET /tools` 端点获取每个工具的参数定义。

### 响应格式

所有工具端点返回统一的 `BaseResponse` 格式：

**成功响应**：

```json
{
  "success": true,
  "data": {
    "content": [
      {
        "type": "text",
        "text": "工具执行结果"
      }
    ]
  },
  "error": null
}
```

**失败响应**：

```json
{
  "success": false,
  "data": null,
  "error": "错误描述信息"
}
```

### 查询可用工具

```bash
curl http://localhost:8000/tools \
  -H "Authorization: Bearer your-secure-token"
```

返回所有已注册的 MCP 工具定义，包含工具名称、描述和参数 Schema。

## 方式三：OpenAPI / Swagger 接入

### Swagger UI

浏览器访问 `http://<host>:<port>/docs`，可交互式浏览和测试所有端点。

点击页面上的 **Authorize** 按钮，输入 Bearer Token 即可认证。

### 获取 OpenAPI Schema

```bash
curl http://localhost:8000/openapi.json
```

可基于此 Schema 自动生成客户端 SDK（支持 TypeScript、Python、Java 等），推荐使用 [OpenAPI Generator](https://openapi-generator.tech/)。

## 辅助端点

### 健康检查

```bash
curl http://localhost:8000/health
```

```json
{"status": "healthy"}
```

适用于负载均衡器或容器编排的健康探针。

### 日志查看

- **Web 界面**: `GET /api/logs` — 内置日志仪表盘，支持按级别过滤、搜索、自动滚动
- **WebSocket**: `ws://<host>:<port>/api/logs/ws` — 实时日志流，连接后先推送最近 1000 条历史日志，之后实时广播新日志

每条日志格式：

```json
{
  "timestamp": "2026-04-12T10:30:00.000000",
  "date": "2026-04-12",
  "level": "INFO",
  "message": "日志内容",
  "id": 42
}
```

### 服务信息

```bash
curl http://localhost:8000/
```

```json
{
  "message": "Automata MCP Server is running",
  "version": "1.0.0",
  "tools_count": 24
}
```

## 环境变量配置

接入前需确保服务端已正确配置以下环境变量：

| 变量 | 必需 | 说明 |
|------|------|------|
| `HOST` | 是 | 服务监听地址 |
| `PORT` | 是 | 服务监听端口 |
| `ALLOWED_ORIGINS` | 是 | CORS 允许的来源，多个用逗号分隔 |
| `ALLOWED_METHODS` | 是 | CORS 允许的 HTTP 方法 |
| `AUTOMATA_ACCESS_TOKEN` | 否* | 访问令牌，未设置时所有请求免认证 |
| `DEBUG` | 否 | 设为 `true` 启用详细错误信息 |

*强烈建议在生产环境中设置 `AUTOMATA_ACCESS_TOKEN`。

## 常见问题

### Q: MCP 客户端连不上服务？

1. 确认服务正在运行：`curl http://<host>:<port>/health`
2. 确认使用正确的端点：MCP 协议使用 `/mcp`，不是 `/sse`
3. 检查防火墙和网络连通性
4. 确认 Access Token 匹配

### Q: 请求返回 401？

1. 确认 `Authorization` 头格式为 `Bearer <token>`（注意 `Bearer` 后有空格）
2. 确认 Token 与服务端 `AUTOMATA_ACCESS_TOKEN` 一致
3. Token 长度需 >= 8 个字符

### Q: 如何知道某个工具需要什么参数？

1. 访问 `/docs` 查看交互式文档
2. 调用 `GET /tools` 查看所有工具的参数 Schema
3. 查看 `docs/plugins.md` 中的工具列表

### Q: MCP 工具名和 REST 端点名不一致？

MCP 工具名来自 OpenAPI Schema 的 `operationId`，由 `fastapi-mcp` 自动生成。REST 端点名来自工具目录名（`tool_name` 字段）。两者可能不同，调用时请以 MCP 客户端展示的工具列表为准。
